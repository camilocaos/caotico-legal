# Politica de Privacidade do Caotico

Data de vigencia: 11 de julho de 2026

Esta Politica de Privacidade explica como o **Caotico**, bot de Discord operado pelo **Caos Labs**, trata dados ao funcionar em servidores Discord.

Esta e uma versao inicial e deve ser atualizada conforme novos recursos forem adicionados.

## 1. Resumo

O Caotico usa apenas os dados necessarios para entregar suas funcionalidades. Nesta versao inicial, o bot nao vende dados, nao cria perfil comercial de usuarios e nao usa conteudo de mensagens para treinar modelos de inteligencia artificial.

## 2. Dados que o Caotico pode acessar

Dependendo das permissoes concedidas no Discord e dos recursos usados, o Caotico pode acessar dados tecnicos como:

- ID do servidor;
- ID de cargos configurados;
- ID de usuarios envolvidos em interacoes com comandos;
- nomes publicos de servidores, cargos e usuarios quando fornecidos pela API do Discord;
- eventos necessarios para funcionalidades, como entrada de novos membros no servidor;
- informacoes tecnicas de interacoes slash, como nome do comando usado e servidor onde foi usado.

## 3. Dados que o Caotico salva

Nesta versao inicial, o Caotico salva configuracoes minimas por servidor, como:

- ID do servidor;
- ID do cargo configurado para autorole.

Esses dados sao salvos para que o bot consiga lembrar qual cargo deve entregar automaticamente quando uma pessoa entra no servidor.

## 4. Dados que o Caotico nao coleta intencionalmente

Nesta versao inicial, o Caotico nao coleta intencionalmente:

- mensagens privadas de usuarios;
- senhas, tokens ou credenciais de usuarios;
- dados bancarios ou financeiros;
- documentos pessoais;
- dados de saude;
- informacoes sensiveis fora do necessario para funcionamento no Discord.

O Caotico nunca pedira senha, token de usuario ou codigo de autenticacao do Discord.

## 5. Como os dados sao usados

Os dados sao usados para:

- executar comandos slash;
- configurar recursos do servidor;
- entregar cargos automaticamente quando o autorole estiver ativo;
- diagnosticar erros tecnicos;
- melhorar a estabilidade e seguranca do bot.

## 6. Compartilhamento de dados

O Caos Labs nao vende dados do Caotico.

Dados tecnicos podem ser processados por servicos necessarios para hospedar, operar ou manter o bot, como plataformas de hospedagem, banco de dados, logs ou ferramentas de monitoramento, quando esses servicos forem adotados.

Tambem podemos divulgar informacoes se isso for necessario para cumprir obrigacoes legais, proteger usuarios, investigar abuso ou responder a solicitacoes validas de autoridades competentes.

## 7. Retencao de dados

Configuracoes do servidor sao mantidas enquanto o bot estiver configurado para aquele servidor ou enquanto forem necessarias para operar o recurso.

Se o bot for removido de um servidor, as configuracoes podem permanecer por um periodo razoavel para fins de recuperacao, suporte e manutencao. Administradores podem solicitar a exclusao dessas configuracoes.

## 8. Exclusao de dados

Administradores de servidor podem remover configuracoes usando comandos do bot, como:

```text
/autorole desativar
```

Para solicitar exclusao de dados relacionados a um servidor, entre em contato com o Caos Labs pelo canal de suporte informado na pagina publica do projeto ou no perfil do aplicativo.

## 9. Seguranca

O Caos Labs busca proteger os dados usados pelo Caotico com medidas razoaveis de seguranca, como manter tokens fora do codigo-fonte publico e limitar o armazenamento aos dados necessarios.

Nenhum sistema e totalmente livre de riscos. Caso seja identificado um incidente relevante de seguranca, medidas apropriadas serao tomadas conforme a situacao.

## 10. Criancas e adolescentes

O Caotico foi criado para funcionar dentro do Discord e deve ser usado de acordo com os requisitos de idade e regras do Discord.

O Caotico nao deve ser direcionado a pessoas abaixo da idade minima permitida pelo Discord ou pela lei aplicavel.

## 11. Mudancas nesta politica

Esta Politica de Privacidade pode ser atualizada conforme o Caotico evolui, especialmente quando novos recursos forem adicionados.

A data de vigencia no topo indica a versao atual.
